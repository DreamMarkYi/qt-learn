# 窗口关闭崩溃 / 卡死 修复说明

## 现象
关闭主窗口时程序崩溃或长时间卡住、无法正常退出。

## 根因分析

### 主因：析构顺序错误（悬空指针）
修复前 `MainWindow::~MainWindow()` 只有一句：

```cpp
MainWindow::~MainWindow()
{
    delete ui;
}
```

`m_capturer` 是用 `new DesktopCapturer(this)` 创建的、以 `MainWindow` 为父对象的子对象。
按 Qt 的对象树规则，子对象要等 `~MainWindow` **函数体执行完**、进入基类 `~QObject`
阶段时才会被销毁。

于是关闭时实际发生的顺序是：

1. `delete ui;` 先执行 —— 连同 `m_previewLabel` 在内的所有 UI 控件被删除。
2. 但此刻**渲染线程仍在以 60fps 运行**：
   `GlCompositor::renderTick()`（`GlCompositor.cpp:140`）
   → `emit frameReady`（`GlCompositor.cpp:192`）
   → `DesktopCapturer` 转发 `emit previewFrame`（`DesktopCapturer.h:62-67`）
   → 跨线程排队投递到 `MainWindow` 的 lambda（`mainwindow.cpp:23-29`）。
3. 该 lambda 里 `m_previewLabel->setPixmap(...)` 访问的已是**悬空指针**
   （label 在第 1 步被删了），→ 崩溃。

即：后台线程还在产帧回调，却先把它要写入的控件拆掉了。

### 次因：BlockingQueuedConnection 的潜在阻塞
`~DesktopCapturer()`（`DesktopCapturer.h:105-119`）用
`Qt::BlockingQueuedConnection` 调用渲染线程的 `cleanup()`。这要求渲染线程的事件
循环此刻空闲。若刚打开窗口就立即关闭，而渲染线程的 `init()`（`GlCompositor.cpp:195`，
内部要加载 Live2D 模型，较慢）尚未跑完，事件循环被占住，主线程会一直阻塞等待，
表现为“点了关闭没反应”。它通常会自行解开，但叠加主因后会放大成“关不掉”。

## 修复内容

**文件：`mainwindow.cpp` —— `MainWindow::~MainWindow()`**

在 `delete ui` **之前**先显式销毁 `m_capturer`：

```cpp
MainWindow::~MainWindow()
{
    // 先停后台管线（渲染线程），再删 UI。
    if (m_capturer) {
        delete m_capturer;     // 其析构 stopAllPreviews/stopPush + 渲染线程 quit()+wait()
        m_capturer = nullptr;  // 等线程彻底停下、不再有 previewFrame 回调后再删 UI
    }
    delete ui;
}
```

`delete m_capturer` 会触发 `~DesktopCapturer()`，它依次：
`stopAllPreviews()` → `stopPush()` → 阻塞调用 `cleanup()` → `m_renderThread->quit()`
→ `m_renderThread->wait()`。`wait()` 返回后渲染线程已结束，不会再发出任何
`frameReady/previewFrame`，此时再 `delete ui` 删除 `m_previewLabel` 就安全了。

> 手动 `delete` 一个子对象是安全的：Qt 会自动把它从父对象的 children 列表里移除，
> 不会在 `~QObject` 阶段被二次删除。

## 修复顺序（关闭时正确的拆解次序）
1. 停止采集与推流：`stopAllPreviews()` / `stopPush()`
2. 停止渲染节拍并释放 GL 资源：`cleanup()`（在渲染线程内，停 `QTimer` + 删纹理/FBO/上下文）
3. 退出并等待渲染线程结束：`m_renderThread->quit()` + `wait()`
4. 删除合成器对象：`delete m_compositor`
5. **最后**才删除 UI：`delete ui`

第 1–4 步由 `delete m_capturer` 一次性触发；本次修复的关键，是把第 5 步明确排到它**之后**。

## 改动文件清单
- `mainwindow.cpp`：`~MainWindow()` 在 `delete ui` 前增加 `delete m_capturer; m_capturer=nullptr;`，并补注释说明顺序约束。
