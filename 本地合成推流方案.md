# 本地合成推流改造方案

把"桌面画面 + 摄像头画面"在 **Qt 本地用 QPainter 合成成一张画面**,再把这张合成帧通过管道喂给 ffmpeg 编码推流。ffmpeg 不再负责抓屏和 overlay 合成,**只当编码器**(rawvideo → libx264 → RTMP)。

## 为什么这样改

| | 现在(ffmpeg 合成) | 本地合成(本方案) |
|---|---|---|
| 抓屏/合成 | ffmpeg(gdigrab + dshow + overlay) | Qt(QScreenCapture + QCamera + QPainter) |
| ffmpeg 角色 | 采集 + 合成 + 编码 | 仅编码 |
| 预览与推流 | 两套独立捕获 | 同一张合成帧,所见即所推 |
| 灵活性 | 只能用 ffmpeg 滤镜 | 任意 QPainter 绘制(水印/文字/转场) |
| CPU/内存 | 低 | 较高(帧在 CPU 层来回拷贝) |

## 数据链路

```
QScreenCapture ─→ QMediaCaptureSession ─→ screenSink ─┐
                                                       ├─ FrameCompositor
QCamera ───────→ QMediaCaptureSession ─→ cameraSink ──┘   (QPainter 合成)
                                                              │ frameReady(QImage)
                                            ┌─────────────────┴─────────────────┐
                                            ▼                                   ▼
                                    QLabel 预览(setPixmap)            FFmpegStreamer::writeFrame
                                                                              │ stdin 管道
                                                                    ffmpeg -f rawvideo -i -  → libx264 → flv → rtmp
```

关键点:**桌面帧驱动合成节奏**(每来一张桌面帧就合成输出一帧),相机帧只缓存最新一张。这样输出帧率稳定 = 桌面捕获帧率,避免两路帧率不一致导致时间线错乱。

## 改动文件清单

| 文件 | 改动 |
|---|---|
| `FrameCompositor.h` | **新增**。两路 sink + QPainter 合成,输出 `frameReady` 信号 |
| `FFmpegStreamer.h` | 重写。改成读 stdin 管道的纯编码器,新增 `writeFrame` |

| `mainwindow.cpp` | 预览绑定改为 QLabel |

## 第 1 步:新增 `FrameCompositor.h`

本地合成器。持有两个 `QVideoSink` 分别接桌面和相机,在桌面帧到达时用 `QPainter` 把相机帧画到右下角,合成结果通过 `frameReady` 抛出。

```cpp
#ifndef FRAMECOMPOSITOR_H
#define FRAMECOMPOSITOR_H

#include <QObject>
#include <QVideoSink>
#include <QVideoFrame>
#include <QImage>
#include <QPainter>
#include <QSize>

// 本地合成器：把桌面帧与相机帧合成为一张画面（画中画），
// 合成结果通过 frameReady 信号抛出，既用于预览也用于推流。
class FrameCompositor : public QObject
{
    Q_OBJECT
public:
    explicit FrameCompositor(QObject *parent = nullptr) : QObject(parent) {
        m_screenSink = new QVideoSink(this);
        m_cameraSink = new QVideoSink(this);

        // 桌面帧驱动整个合成节奏：每来一张桌面帧就合成并输出一帧
        connect(m_screenSink, &QVideoSink::videoFrameChanged,
                this, &FrameCompositor::onScreenFrame);

        // 相机帧只缓存最新一张，不主动触发合成
        connect(m_cameraSink, &QVideoSink::videoFrameChanged, this,
                [this](const QVideoFrame &f) {
            QImage img = f.toImage();
            if (!img.isNull())
                m_lastCamera = img;
        });
    }

    QVideoSink* screenSink() const { return m_screenSink; }
    QVideoSink* cameraSink() const { return m_cameraSink; }

    QSize outputSize() const { return m_outputSize; }

    // 是否把相机画面叠加进合成结果（画中画开关）
    void setOverlay(bool on) { m_overlay = on; }
    bool overlay() const { return m_overlay; }

signals:
    // 合成完成的一帧（固定为 m_outputSize 大小，Format_ARGB32）
    void frameReady(const QImage &frame);

private slots:
    void onScreenFrame(const QVideoFrame &frame) {
        QImage base = frame.toImage();
        if (base.isNull()) return;

        // 1) 统一缩放到输出分辨率，控制推流数据量
        if (base.size() != m_outputSize)
            base = base.scaled(m_outputSize, Qt::IgnoreAspectRatio,
                               Qt::SmoothTransformation);

        // 2) 转成 ARGB32，保证像素字节序为 BGRA（小端），与 ffmpeg 的 bgra 对齐
        if (base.format() != QImage::Format_ARGB32)
            base = base.convertToFormat(QImage::Format_ARGB32);

        // 3) 画中画：把最近一张相机帧画到右下角
        if (m_overlay && !m_lastCamera.isNull()) {
            QImage cam = m_lastCamera.scaled(m_camSize, Qt::KeepAspectRatio,
                                             Qt::SmoothTransformation);
            QPainter p(&base);
            p.drawImage(base.width() - cam.width() - m_margin,
                        base.height() - cam.height() - m_margin, cam);
            p.end();
        }

        emit frameReady(base);
    }

private:
    QVideoSink* m_screenSink = nullptr;
    QVideoSink* m_cameraSink = nullptr;

    QImage m_lastCamera;                    // 最近一张相机帧
    bool   m_overlay = true;                // 是否画中画

    const QSize m_outputSize{1920, 1080};   // 合成/推流输出分辨率
    const QSize m_camSize{480, 360};        // 相机小窗尺寸
    const int   m_margin = 20;              // 距右下角边距(px)
};

#endif // FRAMECOMPOSITOR_H
```

> **为什么固定输出 1920x1080?** 桌面原始分辨率 2560x1440 的 BGRA 单帧约 14MB,25fps 就是 ~350MB/s 的拷贝+写管道压力。降到 1080p 后单帧约 8MB,压力减半,质量基本无损。你可以按需调 `m_outputSize`。

## 第 2 步:重写 `FFmpegStreamer.h`

不再让 ffmpeg 抓屏/合成,改成**从 stdin 读原始帧**的纯编码器。新增 `writeFrame(const QImage&)`,把合成帧的原始像素逐行写进 ffmpeg。

```cpp
#ifndef FFMPEGSTREAMER_H
#define FFMPEGSTREAMER_H

#include <QObject>
#include <QProcess>
#include <QImage>
#include <QSize>
#include <QDebug>

// 纯编码推流器：从 stdin 接收 BGRA 原始帧，用 libx264 编码后推到 RTMP。
class FFmpegStreamer : public QObject
{
    Q_OBJECT
public:
    explicit FFmpegStreamer(QObject *parent = nullptr) : QObject(parent) {
        m_ffmpegProcess = new QProcess(this);
        connect(m_ffmpegProcess, &QProcess::readyReadStandardError, this, [this]() {
            qDebug() << "FFmpeg Log:" << m_ffmpegProcess->readAllStandardError();
        });
    }

    // 🚀 开启推流：size 是输入原始帧的尺寸，fps 是帧率
    void startPush(const QSize &size, int fps = 25) {
        if (m_ffmpegProcess->state() == QProcess::Running) return;

        m_frameSize = size;

        QString aliyunIP = "8.152.169.7";
        QString rtmpUrl = QString("rtmp://%1:1935/live/livestream").arg(aliyunIP);

        QStringList args;
        // —— 输入：从 stdin 读 BGRA 原始帧 ——
        args << "-f" << "rawvideo"
             << "-pixel_format" << "bgra"      // 对应 QImage::Format_ARGB32(小端=BGRA)
             << "-video_size" << QString("%1x%2").arg(size.width()).arg(size.height())
             << "-framerate" << QString::number(fps)
             << "-i" << "-"                     // '-' 表示从标准输入读
        // —— 编码：H.264 ——
             << "-vcodec" << "libx264"
             << "-preset" << "ultrafast"
             << "-tune" << "zerolatency"
             << "-pix_fmt" << "yuv420p"         // 编码输出标准像素格式
        // —— 封装 + 目的地 ——
             << "-f" << "flv"
             << rtmpUrl;

        m_ffmpegProcess->start("ffmpeg", args);
        m_ffmpegProcess->waitForStarted();
        qDebug() << "🚀 [FFmpeg] 纯编码推流已启动" << size << fps << "fps";
    }

    // 写入一帧合成画面。返回 false 表示因背压丢帧或未在推流。
    bool writeFrame(const QImage &src) {
        if (m_ffmpegProcess->state() != QProcess::Running) return false;

        // 尺寸不符直接丢弃，避免花屏/崩溃
        if (src.size() != m_frameSize) return false;

        // 背压保护：写缓冲堆积过多(>1 帧)时丢帧，防止内存暴涨和延迟累积
        const qint64 frameBytes = qint64(m_frameSize.width()) * m_frameSize.height() * 4;
        if (m_ffmpegProcess->bytesToWrite() > frameBytes) return false;

        // 统一为 ARGB32，确保字节序与 bgra 对齐
        QImage img = (src.format() == QImage::Format_ARGB32)
                         ? src : src.convertToFormat(QImage::Format_ARGB32);

        // 逐行写入：QImage 每行可能有对齐填充(bytesPerLine > width*4)，
        // 必须按 width*4 写有效像素，不能直接整块写 sizeInBytes()。
        const int rowBytes = img.width() * 4;
        for (int y = 0; y < img.height(); ++y) {
            m_ffmpegProcess->write(
                reinterpret_cast<const char*>(img.constScanLine(y)), rowBytes);
        }
        return true;
    }

    void stopPush() {
        if (m_ffmpegProcess->state() == QProcess::Running) {
            qDebug() << "⏹️ [FFmpeg] 正在关闭推流器...";
            m_ffmpegProcess->closeWriteChannel();   // 关闭 stdin，ffmpeg 收到 EOF 自然退出
            if (!m_ffmpegProcess->waitForFinished(5000))
                m_ffmpegProcess->kill();
            qDebug() << "⏹️ [FFmpeg] 推流已断开";
        }
    }

    bool isPushing() const {
        return m_ffmpegProcess->state() == QProcess::Running;
    }

    ~FFmpegStreamer() { stopPush(); }

private:
    QProcess* m_ffmpegProcess = nullptr;
    QSize     m_frameSize;
};

#endif // FFMPEGSTREAMER_H
```

> **逐行写 vs 整块写:** `QImage` 每行字节数 `bytesPerLine()` 可能因 4 字节对齐而大于 `width*4`(有填充)。1920 宽 ×4=7680 恰好对齐,整块写也行;但逐行写对任意宽度都安全,是稳妥写法。
>
> **关 stdin 退出:** 用 `closeWriteChannel()` 给 ffmpeg 送 EOF,比原来 `write("q\n")` 更适合管道输入模式。

## 第 4 步:重写 `DesktopCapturer.h`

桌面捕获输出从 widget 改到合成器的 `screenSink`,相机输出到 `cameraSink`。推流改成把合成帧喂给 ffmpeg。预览不再用 `QVideoWidget`(它只显示不给帧),改成把 `frameReady` 的合成帧抛给 UI。

```cpp
#ifndef DESKTOPCAPTURER_H
#define DESKTOPCAPTURER_H

#include <QObject>
#include <QScreenCapture>
#include <QMediaCaptureSession>
#include <QGuiApplication>
#include <QImage>
#include <QDebug>

#include "CameraManager.h"
#include "FFmpegStreamer.h"
#include "FrameCompositor.h"

class DesktopCapturer : public QObject
{
    Q_OBJECT
public:
    explicit DesktopCapturer(QObject *parent = nullptr) : QObject(parent) {
        // 1. 桌面捕获链路
        m_screenSession = new QMediaCaptureSession(this);
        m_screenCapturer = new QScreenCapture(this);
        m_screenCapturer->setScreen(QGuiApplication::primaryScreen());
        m_screenSession->setScreenCapture(m_screenCapturer);

        // 2. 相机管理器
        m_cameraManager = new CameraManager(this);

        // 3. 推流器
        m_streamer = new FFmpegStreamer(this);

        // 4. 本地合成器，并把两路捕获接到它的 sink 上
        m_compositor = new FrameCompositor(this);
        m_screenSession->setVideoSink(m_compositor->screenSink());
        m_cameraManager->setVideoSink(m_compositor->cameraSink());

        // 5. 合成帧出炉：一路预览(转发给 UI)，一路推流
        connect(m_compositor, &FrameCompositor::frameReady, this,
                [this](const QImage &frame) {
            emit previewFrame(frame);            // 给 UI 预览
            if (m_streamer->isPushing())
                m_streamer->writeFrame(frame);   // 给 ffmpeg 推流
        });
    }

    // 🎬 开启本地双路预览（合成帧会通过 previewFrame 信号持续吐出）
    void startAllPreviews() {
        m_screenCapturer->start();
        m_cameraManager->start();
        qDebug() << "▶️ [总控] 本地双路预览已启动";
    }

    void stopAllPreviews() {
        m_screenCapturer->stop();
        m_cameraManager->stop();
        qDebug() << "⏹️ [总控] 本地预览已关闭";
    }

    // 🚀 开启推流：用合成器当前的输出尺寸告诉 ffmpeg
    void startPush() {
        m_streamer->startPush(m_compositor->outputSize(), 25);
    }

    void stopPush() { m_streamer->stopPush(); }
    bool isPushing() const { return m_streamer->isPushing(); }

    // 画中画开关（合成时是否叠加相机）
    void setCameraOverlay(bool on) { m_compositor->setOverlay(on); }
    bool cameraOverlay() const { return m_compositor->overlay(); }

    // 切换相机硬件
    void setCameraDevice(const QCameraDevice &device) {
        m_cameraManager->changeCameraDevice(device);
    }

    static QList<QCameraDevice> availableCameras() {
        return CameraManager::availableCameras();
    }

    ~DesktopCapturer() {
        stopAllPreviews();
        stopPush();
    }

signals:
    // 合成好的预览帧
    void previewFrame(const QImage &frame);

private:
    QScreenCapture* m_screenCapturer = nullptr;
    QMediaCaptureSession* m_screenSession = nullptr;
    CameraManager* m_cameraManager = nullptr;
    FFmpegStreamer* m_streamer = nullptr;
    FrameCompositor* m_compositor = nullptr;
};

#endif // DESKTOPCAPTURER_H
```

> 相比原版:删掉了 `setVideoOutputs`(不再往 widget 输出)、`startPush(bool)` 参数(画中画开关挪到 `setCameraOverlay`)、以及 `setCameraDevice` 里的自动重启推流逻辑(本地合成下换相机不影响 ffmpeg 输入,无需重启)。

## 第 6 步:改 `mainwindow.cpp`

把预览绑定从 `setVideoOutputs` 改为"创建 QLabel + 接 `previewFrame` 信号"。其余按钮逻辑微调:推流不再带参数,画中画开关调 `setCameraOverlay`。

```cpp
#include "mainwindow.h"
#include "./ui_mainwindow.h"
#include <QCameraDevice>
#include <QPixmap>

MainWindow::MainWindow(QWidget *parent)
    : QMainWindow(parent)
    , ui(new Ui::MainWindow)
{
    ui->setupUi(this);

    m_capturer = new DesktopCapturer(this);

    // 在 UI 原来那个 widget 的位置上盖一个 QLabel 当预览画布
    m_previewLabel = new QLabel(ui->widget->parentWidget());
    m_previewLabel->setGeometry(ui->widget->geometry());
    m_previewLabel->setStyleSheet("background:#000;");
    m_previewLabel->setAlignment(Qt::AlignCenter);
    m_previewLabel->show();
    ui->widget->hide();   // 藏掉原来的 QVideoWidget

    // 合成帧到达 → 缩放铺满 label 显示
    connect(m_capturer, &DesktopCapturer::previewFrame, this,
            [this](const QImage &frame) {
        m_previewLabel->setPixmap(
            QPixmap::fromImage(frame).scaled(m_previewLabel->size(),
                                             Qt::KeepAspectRatio,
                                             Qt::SmoothTransformation));
    });

    // 摄像头下拉框
    const auto cams = DesktopCapturer::availableCameras();
    if (cams.isEmpty()) {
        ui->cameraCombo->addItem("（未检测到摄像头）");
        ui->cameraCombo->setEnabled(false);
    } else {
        for (const QCameraDevice &c : cams)
            ui->cameraCombo->addItem(c.description(), c.id());
    }
    connect(ui->cameraCombo, QOverload<int>::of(&QComboBox::currentIndexChanged),
            this, [this](int idx) {
        const auto cams = DesktopCapturer::availableCameras();
        if (idx < 0 || idx >= cams.size()) return;
        m_capturer->setCameraDevice(cams.at(idx));
    });

    // 【开启/关闭 预览】
    connect(ui->captureBtn, &QPushButton::clicked, this, [this]() {
        static bool previewing = false;
        if (!previewing) {
            m_capturer->startAllPreviews();
            ui->captureBtn->setText("关闭预览");
        } else {
            m_capturer->stopAllPreviews();
            ui->captureBtn->setText("开启预览");
        }
        previewing = !previewing;
    });

    // 【开始/停止 推流】
    connect(ui->pushBtn, &QPushButton::clicked, this, [this]() {
        static bool pushing = false;
        if (!pushing) {
            m_capturer->startPush();
            ui->pushBtn->setText("停止推流");
        } else {
            m_capturer->stopPush();
            ui->pushBtn->setText("开始推流");
        }
        pushing = !pushing;
    });

    // 【画中画开关】
    ui->switchBtn->setText(m_capturer->cameraOverlay() ? "推流含相机" : "仅推流桌面");
    connect(ui->switchBtn, &QPushButton::clicked, this, [this]() {
        bool on = !m_capturer->cameraOverlay();
        m_capturer->setCameraOverlay(on);
        ui->switchBtn->setText(on ? "推流含相机" : "仅推流桌面");
    });
}

MainWindow::~MainWindow()
{
    delete ui;
}
```

> 画中画开关现在是**实时生效**的:`setCameraOverlay` 改的是合成器的标志位,下一帧合成立刻按新设置画或不画,不用像之前那样重启 ffmpeg。

## 常见坑速查

1. **花屏/绿屏** — 几乎都是像素格式或尺寸不匹配。确认三处一致:`QImage` 是 `Format_ARGB32`、ffmpeg 参数是 `bgra`、`-video_size` 等于实际帧尺寸。
2. **ffmpeg 起不来或秒退** — 在命令行先手动跑一遍 `ffmpeg -f rawvideo -pixel_format bgra -video_size 1920x1080 -framerate 25 -i - ...`,看 `readyReadStandardError` 打出来的日志定位。
3. **CPU 飙高/卡顿** — 调小 `FrameCompositor::m_outputSize`(如 1280x720),或把 `SmoothTransformation` 换成 `FastTransformation`。
4. **延迟越来越大** — 背压没控住。`FFmpegStreamer::writeFrame` 里的 `bytesToWrite()` 丢帧逻辑就是干这个的,阈值可调更激进。
5. **预览不动** — 检查 `previewFrame` 信号名在 signals 声明、emit、connect 三处完全一致;以及确实调用了 `startAllPreviews()`。
6. **`toImage()` 返回空** — 某些后端首帧或切换设备瞬间会给空帧,代码里都已 `isNull()` 判空跳过,属正常。

## 改动影响范围小结

- **删除/弃用**:`DesktopCapturer::setVideoOutputs`、`startPush(bool)`、FFmpeg 的 `gdigrab/dshow/overlay` 整套采集合成参数。
- **保留**:RTMP 地址、libx264 编码参数、相机切换、下拉框逻辑。
- **ffmpeg 依赖仍然需要**:它仍是编码+推流的执行者,只是不再负责采集和合成。
