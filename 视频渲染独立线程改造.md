# 把视频渲染（GlCompositor）改成独立线程

## 1. 为什么要改 / 改哪里

当前所有工作都在 GUI 主线程上完成。帧链路：

```
QScreenCapture ──► screenSink ──► GlCompositor::onScreenFrame
                                       │  makeCurrent + 上传纹理 + drawQuad
                                       │  glFinish()          ← 阻塞等 GPU
                                       │  m_fbo->toImage()    ← GPU 回读(最慢)
                                       ▼
                                  emit frameReady(QImage)
                                       │
                          DesktopCapturer 的 lambda
                              ├─ previewFrame → setPixmap (UI)
                              └─ writeFrame   → ffmpeg stdin (管道)
```

卡顿来自 `GlCompositor::onScreenFrame` 里的 `glFinish()` + `m_fbo->toImage()`
（GPU 回读），每帧都在主线程同步阻塞，分辨率越高越明显。

**改造目标**：把整个 `GlCompositor` 渲染对象搬到一个专用的 `QThread`，
GUI 线程只负责接收最终 `QImage` 去预览，不再被 GPU 回读拖住。

改造后链路：

```
[渲染线程]                              [主线程]
QVideoSink(帧到达, 队列投递)
   └─ onScreenFrame (GL 渲染 + 回读)
          └─ emit frameReady ──(跨线程队列连接)──► previewFrame → setPixmap
                                                  └─ writeFrame → ffmpeg
```

## 2. 三个必须遵守的约束

1. **OpenGL 上下文必须在"将来用它的线程"里创建**。
   所以构造函数里不能再 `new QOpenGLContext` / `makeCurrent` / `buildGl`，
   要全部推迟到一个 `init()` 槽，由 `QThread::started` 在工作线程里触发。
2. **`moveToThread` 要求对象没有 parent**。
   `GlCompositor` 不能再写成 `new GlCompositor(this)`，必须无父对象创建，
   生命周期由 `DesktopCapturer` 手动管理。
3. **GL 资源（纹理 / FBO / 着色器）必须在工作线程、上下文 current 时销毁**。
   不能依赖析构函数（它可能在主线程跑），需要一个 `cleanup()` 槽，
   在线程退出前用 `BlockingQueuedConnection` 调一次。

## 3. 改动文件清单

| 文件 | 改动 |
|------|------|
| `GlCompositor.h`  | 构造函数瘦身；新增 `init()` / `cleanup()` 槽 |
| `GlCompositor.cpp`| 实现 `init()`（原构造里的 GL 初始化搬来）/ `cleanup()` |
| `DesktopCapturer.h`| 新增 `QThread`；`moveToThread`；启动/关闭线程 |

`onScreenFrame`、`uploadFrame`、`drawQuad`、`buildGl` 的函数体**不用改**，
它们本来就自带 `makeCurrent`/`doneCurrent`，搬到工作线程照样工作。

## 4. GlCompositor.h（完整改造后）

关键点：构造函数只 new 出 sink，**不碰 GL**；GL 初始化全部移到 `init()`；
新增 `cleanup()` 销毁 GL 资源。注意 `m_ctx`/`m_surface` 的 parent 也要去掉
（它们会跟着对象一起 moveToThread，不能让 Qt 在主线程析构时去碰 GL）。

```cpp
#ifndef GLCOMPOSITOR_H
#define GLCOMPOSITOR_H

#include <QObject>
#include <QVideoSink>
#include <QVideoFrame>
#include <QImage>
#include <QSize>
#include <QOpenGLContext>
#include <QOffscreenSurface>
#include <QOpenGLFunctions_3_3_Core>
#include <QOpenGLShaderProgram>
#include <QOpenGLFramebufferObject>
#include <QOpenGLBuffer>
#include <QOpenGLVertexArrayObject>
#include <QDebug>

class GlCompositor : public QObject, protected QOpenGLFunctions_3_3_Core
{
    Q_OBJECT
public:
    // 构造函数瘦身：只建 sink，绝不碰 OpenGL。
    // 不传 parent，方便后续 moveToThread。
    explicit GlCompositor(QObject *parent = nullptr) : QObject(parent) {
        m_screenSink = new QVideoSink(this);
        m_cameraSink = new QVideoSink(this);

        // sink 的信号槽是 Qt::AutoConnection：发送方(媒体线程)与接收方
        // (本对象所在的渲染线程)不同线程时，自动变成队列连接，线程安全。
        connect(m_screenSink, &QVideoSink::videoFrameChanged,
                this, &GlCompositor::onScreenFrame);
        connect(m_cameraSink, &QVideoSink::videoFrameChanged, this,
                [this](const QVideoFrame &f) { m_lastCamera = f; });
    }

    QVideoSink* screenSink() const { return m_screenSink; }
    QVideoSink* cameraSink() const { return m_cameraSink; }
    QSize outputSize() const { return m_outputSize; }
    void setOverlay(bool on) { m_overlay = on; }
    bool overlay() const { return m_overlay; }

public slots:
    // 在渲染线程里执行：创建上下文 + 离屏面 + 编译着色器。
    // 由 QThread::started 触发，保证 GL 上下文归属工作线程。
    void init();

    // 在渲染线程里执行：上下文 current 状态下释放所有 GL 资源。
    // 线程退出前用 BlockingQueuedConnection 调一次。
    void cleanup();

signals:
    void frameReady(const QImage &frame);

private:
    QOpenGLContext*  m_ctx = nullptr;
    QOffscreenSurface* m_surface = nullptr;
    QOpenGLShaderProgram* m_prog = nullptr;
    QOpenGLFramebufferObject* m_fbo = nullptr;
    QOpenGLVertexArrayObject m_vao;
    QOpenGLBuffer m_vbo{QOpenGLBuffer::VertexBuffer};
    QOpenGLBuffer m_ebo{QOpenGLBuffer::IndexBuffer};
    QVideoSink* m_screenSink = nullptr;
    QVideoSink* m_cameraSink = nullptr;
    QVideoFrame m_lastCamera;

    bool m_overlay = true;
    const QSize m_outputSize{1920, 1080};
    const QSize m_camSize{480, 360};
    const int   m_margin = 20;

    GLuint m_texScreen = 0;
    GLuint m_texCamera = 0;
    QSize  m_texScreenSize;
    QSize  m_texCameraSize;

    void buildGl();
    void onScreenFrame(const QVideoFrame &screenFrame);
    bool uploadFrame(GLuint &tex, QSize &cachedSize, const QVideoFrame &f);
    void drawQuad(GLuint tex, float x, float y, float w, float h);
};

#endif // GLCOMPOSITOR_H
```

## 5. GlCompositor.cpp（新增 init / cleanup，其余不变）

把**原来构造函数里**的那段 GL 初始化（创建上下文、离屏面、`makeCurrent`、
`initializeOpenGLFunctions`、`buildGl`）整体搬进 `init()`。
`buildGl()`、`drawQuad()`、`uploadFrame()`、`onScreenFrame()` 四个函数
**一字不改**，沿用现有实现。

在文件顶部、`buildGl()` 之前新增两个函数：

```cpp
#include "GlCompositor.h"

// ── 在渲染线程里初始化 OpenGL（原构造函数里的 GL 部分搬到这里）──
void GlCompositor::init()
{
    QSurfaceFormat fmt;
    fmt.setRenderableType(QSurfaceFormat::OpenGL);
    fmt.setVersion(3, 3);
    fmt.setProfile(QSurfaceFormat::CoreProfile);

    // 上下文不设 parent：它属于当前线程，由 cleanup() 显式删除。
    m_ctx = new QOpenGLContext();
    m_ctx->setFormat(fmt);
    if (!m_ctx->create()) {
        qFatal("错误：无法创建需要的 OpenGL 上下文！请检查显卡驱动。");
    }
    qDebug() << "成功创建 OpenGL 上下文，实际版本为:"
             << m_ctx->format().version()
             << " 线程:" << QThread::currentThread();

    m_surface = new QOffscreenSurface();   // 同样不设 parent
    m_surface->setFormat(fmt);
    m_surface->create();

    m_ctx->makeCurrent(m_surface);
    initializeOpenGLFunctions();
    buildGl();
    m_ctx->doneCurrent();
}

// ── 在渲染线程里释放所有 GL 资源（上下文 current 状态下执行）──
void GlCompositor::cleanup()
{
    if (!m_ctx) return;
    m_ctx->makeCurrent(m_surface);

    if (m_texScreen) { glDeleteTextures(1, &m_texScreen); m_texScreen = 0; }
    if (m_texCamera) { glDeleteTextures(1, &m_texCamera); m_texCamera = 0; }

    m_vao.destroy();
    m_vbo.destroy();
    m_ebo.destroy();
    delete m_prog;  m_prog = nullptr;
    delete m_fbo;   m_fbo  = nullptr;

    m_ctx->doneCurrent();

    delete m_surface; m_surface = nullptr;
    delete m_ctx;     m_ctx     = nullptr;
    qDebug() << "🧹 [GL] 渲染资源已在工作线程释放";
}

// 下面 buildGl() / drawQuad() / uploadFrame() / onScreenFrame()
// 保持原样，无需改动……
```

> 记得在 `GlCompositor.cpp` 顶部补一个 `#include <QThread>`
> （`init()` 里用 `QThread::currentThread()` 打日志；不打日志可省）。

## 6. DesktopCapturer.h（完整改造后）

核心三步：
1. `m_compositor = new GlCompositor()` **不传 parent**（moveToThread 要求）。
2. `m_compositor->moveToThread(m_renderThread)`，再 `connect(thread, started, comp, init)`，
   保证 GL 上下文在工作线程里创建。
3. 析构时先 `BlockingQueuedConnection` 调 `cleanup()`，再 `quit()` + `wait()`，
   最后手动 `delete m_compositor`（它没 parent，不会被自动回收）。

`frameReady → previewFrame / writeFrame` 这条连接是**跨线程**的：
`GlCompositor` 在渲染线程发信号，`DesktopCapturer` 在主线程，
`Qt::AutoConnection` 自动用队列连接，`QImage` 是隐式共享、可安全跨线程拷贝。

```cpp
#ifndef DESKTOPCAPTURER_H
#define DESKTOPCAPTURER_H

#include <QObject>
#include <QScreenCapture>
#include <QMediaCaptureSession>
#include <QGuiApplication>
#include <QThread>
#include <QImage>
#include <QDebug>

#include "CameraManager.h"
#include "FFmpegStreamer.h"
#include "GlCompositor.h"

class DesktopCapturer : public QObject
{
    Q_OBJECT
public:
    explicit DesktopCapturer(QObject *parent = nullptr) : QObject(parent) {
        // 1. 桌面捕获链路
        m_screenSession = new QMediaCaptureSession(this);
        m_screenCapturer = new QScreenCapture(this);
        m_screenCapturer->setScreen(QGuiApplication::primaryScreen());
        m_screenSession->setScreenCapture(m_screenCapturer);

        // 2. 相机管理器
        m_cameraManager = new CameraManager(this);

        // 3. 推流器（纯编码，吃 stdin 原始帧）
        m_streamer = new FFmpegStreamer(this);

        // 4. GPU 合成器：⚠️ 不传 parent，才能 moveToThread
        m_compositor = new GlCompositor();          // 注意：没有 this
        m_renderThread = new QThread(this);
        m_compositor->moveToThread(m_renderThread);

        // 线程一启动就在工作线程里初始化 GL 上下文
        connect(m_renderThread, &QThread::started,
                m_compositor, &GlCompositor::init);
        m_renderThread->start();

        // sink 在主线程拿到指针即可，videoFrameChanged 会队列投递到渲染线程
        m_screenSession->setVideoSink(m_compositor->screenSink());
        m_cameraManager->setVideoSink(m_compositor->cameraSink());

        // 5. 合成帧出炉（跨线程队列连接）：一路预览，一路推流
        connect(m_compositor, &GlCompositor::frameReady, this,
                [this](const QImage &frame) {
            emit previewFrame(frame);            // 给 UI 预览
            if (m_streamer->isPushing())
                m_streamer->writeFrame(frame);   // 给 ffmpeg 推流
        });
    }

    // 🎬 开启本地双路预览
    void startAllPreviews() {
        m_screenCapturer->start();
        m_cameraManager->start();
        qDebug() << "▶️ [总控] 本地双路预览已启动";
    }

    void stopAllPreviews() {
        m_screenCapturer->stop();
        m_cameraManager->stop();
        qDebug() << "⏹️ [总控] 本地预览已关闭";
    }

    void startPush() {
        m_streamer->startPush(m_compositor->outputSize(), 25);
    }
    void stopPush() { m_streamer->stopPush(); }
    bool isPushing() const { return m_streamer->isPushing(); }

    void setCameraOverlay(bool on) { m_compositor->setOverlay(on); }
    bool cameraOverlay() const { return m_compositor->overlay(); }

    void setCameraDevice(const QCameraDevice &device) {
        m_cameraManager->changeCameraDevice(device);
    }

    static QList<QCameraDevice> availableCameras() {
        return CameraManager::availableCameras();
    }

    ~DesktopCapturer() {
        stopAllPreviews();
        stopPush();

        // 关线程顺序很关键：
        // ① 先在工作线程里同步销毁 GL 资源（必须 Blocking，等它真正跑完）
        if (m_renderThread->isRunning()) {
            QMetaObject::invokeMethod(m_compositor, "cleanup",
                                      Qt::BlockingQueuedConnection);
        }
        // ② 退出事件循环并等线程结束
        m_renderThread->quit();
        m_renderThread->wait();
        // ③ compositor 没 parent，手动删（此时已在主线程，且 GL 已清空）
        delete m_compositor;
        m_compositor = nullptr;
    }

signals:
    void previewFrame(const QImage &frame);

private:
    QScreenCapture* m_screenCapturer = nullptr;
    QMediaCaptureSession* m_screenSession = nullptr;
    CameraManager* m_cameraManager = nullptr;
    FFmpegStreamer* m_streamer = nullptr;
    GlCompositor* m_compositor = nullptr;     // ⚠️ 无 parent，手动管理
    QThread* m_renderThread = nullptr;
};

#endif // DESKTOPCAPTURER_H
```

## 7. main.cpp / mainwindow.cpp 要不要改？

不用改。`QImage` 跨线程通过 `frameReady` 队列连接传到主线程，
`mainwindow.cpp` 里的 `setPixmap` 仍在主线程执行，UI 安全。

唯一隐性要求：跨线程信号传 `QImage` 时 Qt 需要该类型已注册。
`QImage` 是 Qt 内建元类型，**默认已注册**，无需 `qRegisterMetaType`。
如果将来改成传自定义结构体，才需要在 `main()` 里注册。

## 8. 验证与排错

- 启动看日志：`成功创建 OpenGL 上下文 … 线程: QThread(0x…)`，
  线程地址应**不同于**主线程地址，说明 GL 跑在工作线程。
- 预览流畅、CPU/GPU 回读不再卡主线程即为成功。
- 退出无 `QOpenGLContext: Moving to thread…` 警告、无崩溃，说明 cleanup 正确。

常见坑：
1. **构造里残留 GL 调用** → 上下文会建在主线程，moveToThread 后报
   `Cannot make QOpenGLContext current in a different thread`。确保全挪到 `init()`。
2. **给 GlCompositor 传了 parent** → `moveToThread` 失败并告警。必须无 parent。
3. **析构没调 cleanup 直接 quit** → 资源在错误线程析构，可能崩溃或泄漏。
   顺序固定为 cleanup(Blocking) → quit → wait → delete。
4. **sink 指针在主线程取没问题**，但不要在主线程直接调 compositor 的渲染方法；
   `setOverlay` 只是写一个 bool，跨线程读写基本无害，要求严格可改成
   `QMetaObject::invokeMethod` 投递。

## 9. 改动速查（照着这个顺序动手）

1. `GlCompositor.h`：构造函数删掉所有 GL 代码，只留 sink 的 new 和 connect；
   新增 `public slots: void init(); void cleanup();`。
2. `GlCompositor.cpp`：顶部加 `#include <QThread>`；新增 `init()`（粘原构造里
   的 GL 初始化）和 `cleanup()`；其余四个函数不动。
3. `DesktopCapturer.h`：加 `#include <QThread>`；`m_compositor = new GlCompositor()`
   去掉 `this`；加 `m_renderThread` 并 `moveToThread` + `started→init` + `start()`；
   析构补 cleanup(Blocking)→quit→wait→delete；成员加 `QThread* m_renderThread`。
4. `main.cpp` / `mainwindow.cpp`：无需改动。

## 10. 数据流向（改造后一图速览）

```
[媒体线程]            [渲染线程 m_renderThread]              [主线程 GUI]
QScreenCapture
   └─videoFrameChanged ─队列─► onScreenFrame
                                 GL 上传/绘制/回读
                                 emit frameReady ─队列─► previewFrame → setPixmap
                                                          └─ writeFrame → ffmpeg(管道)
QCamera
   └─videoFrameChanged ─队列─► m_lastCamera = f
```

三类线程职责清晰：媒体线程产帧、渲染线程做 GL 合成与回读（原来卡主线程的活）、
主线程只做轻量的 UI 刷新和推流写管道。
